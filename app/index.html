Data Collection

The evolution of technology has led to an exponential increase in the amount of information produced per year. Virtually every activity we are involved in translates to some form of big data generated nowadays. 5 exabytes of data is produced everyday and this data ranges from queryable info like credit card transactions, to non queryable ones like videos from surgeries and so on.

Data can be collected from offline sources such as credit card transactions or from the internet by web crawlers such as those used by Google. In a study by Krux in 2012, it was observed that there was a 400% increase in data collection events per page in comparison to 2011, indicating an increase in interest in this data by many companies.

Some of these companies include the aforementioned Google, Social media sites like Facebook and Twitter, which make use of the personal data of useres to explore trends. Other companies (via third party applications) can also make use of this abundance of data for various purposes eg. QuizUp which connects to facebook and has an extensive database of players and favourite topics.

These companies return this data to their local data warehouses for specific analysis, as big data analysis has been shown to be greatly beneficial to business growth and expansion, irrespective of the business size. Take google ad services for instance, which has analysed user search patterns to return personalised adverts.

Storage

Some examples of the way Big Data is stored.


Object-Based Storage

Object based storage works with a flat architecture where data is stored in buckets. The objects are not organised in a hierarchy like files. They exist at the same level in a flat address space called a storage pool and have their own identifiers. This makes it faster and easier to retrieve data rather than having to delve into deep files structures. Object based storage makes use of metadata.

Metadata is a basic piece of information about data that make it easy to find and work with particular attributes of data. Object based storage helps to connect the dots and show relationships between metadata rather than the contents itself. For example it's not as important to know which people visited a particular website as the number of people who came there after a certain article was posted.



Scale-out NAS

The main reason companies with huge data use Scale-out NAS is because of their potentially limitless expansion while still being managed as a single storage. It is a dedicated hard disk storage device and provides a file based storage system. It is a system that can run simultaneously with different NAS nodes. It makes a copy of the data as backup in the background and has fault tolerance. This means that if one or more nodes fail to work, it will not affect the system without any data loss. Advantages of scale out NAS - cost containment.



Hyperscale Storage

Companies that use hyperscale storage include Google and Facebook. Hyperscale is a system that is capable of growing from really small to very large in a very quick and efficient manner. The amount of data stored in a hyperscale system is usually in terms of petabytes. There are open-source platforms that offer data storage and services in this environment such as Hadoop, Ceph, Cassandra. In hyperscale computing, array storage networks are replace with localy connected and network connected storage. It includes very important features that improves performance and high availability.

Map Reduce and Pig Latin

Map Reduce

MapReduce is a programming model and an associated implementation for processing and generating large data sets. Users specify a map function that processes a key/value pair to generate a set of intermediate key/value pairs, and a reduce function that merges all intermediate values associated with the same intermediate key. Many real world tasks are expressible in this model.

Programs written in this functional style are automatically parallelized and executed on a large cluster of commodity machines. The run-time system takes care of the details of partitioning the input data. This allows programmers without any experience with parallel and distributed systems to easily utilize the resources of a large distributed system.

"Map" step: The master node takes the input, divides it into smaller sub-problems, and distributes them to worker nodes. A worker node may do this again in turn, leading to a multi-level tree structure. The worker node processes the smaller problem, and passes the answer back to its master node.

"Reduce" step: The master node then collects the answers to all the sub-problems and combines them in some way to form the output – the answer to the problem it was originally trying to solve.


Uses

MapReduce is useful in a wide range of applications. For example, at Google, MapReduce was used to completely regenerate Google's index of the World Wide Web. It replaced the old ad hoc programs that updated the index and ran the various analyses.


Criticism

MapReduce tasks must be written as acyclic dataflow programs, i.e. a stateless mapper followed by a stateless reducer, that are executed by a batch job scheduler.


About Pig and PigLatin

Pig was initially developed at Yahoo! to allow people using Hadoop® to focus more on analyzing large data sets and spend less time having to write mapper and reducer programs. Like actual pigs, who eat almost anything, the Pig programming language is designed to handle any kind of data—hence the name!

Pig is made up of two components: the first is the language itself, which is called PigLatin and the second is a runtime environment where PigLatin programs are executed, like Eclipse and a Java application. Here, we’ll just refer to the whole entity as Pig.


The Programming Language

Let’s first look at the programming language itself so that you can see how it’s significantly easier than having to write mapper and reducer programs.

The first step in a Pig program is to LOAD the data you want to manipulate from HDFS(Hadoop Distributed File System).

Then you run the data through a set of transformations (which, under the covers, are translated into a set of mapper and reducer tasks)

Finally, you DUMP the data to the screen or you STORE the results in a file somewhere.

Visualisation

How is it different from traditional analytics?


Volume

Due to the sheer volume of Big Data, it is sometimes impossible or too computationally expensive to visualise every single piece of information stored. Data scientists have hence developed visual query languages that make use of datastore statistics to limit the search space to only those areas that potentially contain interesting data.


Velocity

As information is generated at an increasing rate, it becomes increasingly important for data scientists to capture and analyse data in real time so that they will be able to identify and react immediately to rising trends. However, much of the generated data is noise which forms a huge bottleneck that chokes the effectiveness of existing visualisation techniques. A different approach is hence required to efficiently filter and discard tonnes of meaningless information so that the more interesting bits could be added to the visualisation as they arrive.


Variety

Unlike relational databases that only store data that can be easily turned into tabular form, Big Data comes in a wide variety of format. Videos, sounds, and images are all examples of non-tabular that have to be analysed in the Big Data world. The ability to extract interesting patterns and features from these new sources of information forms the core of Big Data Visualisation.
