STORAGE

Some examples of the way Big data is stored.


Object-Based Storage

Object based storage works with a flat architecture where data is stored in buckets. The objects are not organised in a hierarchy like files. They exist at the same level in a flat address space called a storage pool and have their own identifiers. This makes it faster and easier to retrieve data rather than having to delve into deep files structures. Object based storage makes use of metadata.

Metadata is a basic piece of information about data that make it easy to find and work with particular attributes of data. Object based storage helps to connect the dots and show relationships between metadata rather than the contents itself. For example it's not as important to know which people visited a particular website as the number of people who came there after a certain article was posted.



Scale-out NAS

The main reason companies with huge data use Scale-out NAS is because of their potentially limitless expansion while still being managed as a single storage. It is a dedicated hard disk storage device and provides a file based storage system. It is a system that can run simultaneously with different NAS nodes. It makes a copy of the data as backup in the background and has fault tolerance. This means that if one or more nodes fail to work, it will not affect the system without any data loss. Advantages of scale out NAS - cost containment.
